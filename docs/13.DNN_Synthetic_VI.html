1 Overview

This script uses a deep neural network (DNN) to derive data-driven vegetation indices from raw multispectral drone imagery for the NY Winter Malting Barley breeding programme.

1.1 What it does

Standard vegetation indices (NDVI, NDRE, etc.) are fixed, hand-crafted combinations of spectral bands designed for generic vegetation monitoring. Here, we instead let the data determine the optimal band combination for each agronomic or malt-quality trait. The result is a trait-specific Synthetic VI — a scalar value computed from the five raw bands (blue, green, red, red_edge, nir) that maximises predictive accuracy for that trait.

A separate model is trained for each phenotype trait, yielding a distinct Synthetic VI per trait. After model training, the learned encoder weights are extracted and presented as an explicit mathematical formula so the index can be computed directly from band reflectances without re-running the DNN.

1.2 Modelling approach

The DNN has a bottleneck architecture with two components:

  1. Band Encoder (5 → 16 → 8 → 1): three fully-connected layers with ReLU activations compress the five bands into a single scalar — the Synthetic VI. The final layer has no activation, so the index spans the full real line.

  2. Prediction Head (1 → 8 → 1): maps the Synthetic VI through one hidden layer to the trait value.

The bottleneck forces all spectral information through a single neuron before prediction, mimicking the structure of hand-crafted indices (e.g. NDVI uses a single derived value) while allowing non-linear, data-optimised band combinations. Only five specific Timepoint × Environment groups are used, one per trial year, representing the flight timing judged most informative per environment: HELF24_TP7, KET21_TP4, MCG23_TP5, MCG25_TP12, SNY22_TP6.

1.3 Cross-validation

Two complementary CV scenarios are implemented. In both, splits are by GID (genotype ID) to prevent data leakage, and scalers are fit on training data only then applied to test folds.

1.3.1 Leave-One-Environment-Out (LOEO)

All observations from one environment are withheld; the model trains on the remaining four. This measures cross-environment generalisation.

Fold Training environments Validation environment

1.3.2 Within-Environment 5-Fold CV (by GID)

Within each environment, GIDs are shuffled and assigned to 5 folds. The model trains on 4/5 of GIDs and predicts the remaining 1/5. This measures within-environment predictive ability for unseen genotypes.

Each scenario is repeated 3 times with different random seeds. Performance is Pearson r and RMSE on the back-transformed trait scale, benchmarked against linear regression on standard VIs under the same schemes.

4 DNN Architecture

The network has two components trained within each Timepoint × Environment group:

  1. Band Encoder — three fully-connected layers (n_bands → 16 → 8 → 1). The single output neuron is the Synthetic VI — no activation, so it spans the full real line.

  2. Prediction Head — maps the scalar Synthetic VI through one hidden layer to the trait prediction.

Because modelling is within-group, between-year and between-trial offsets are irrelevant; no environment embedding is needed.

# ══════════════════════════════════════════════════════════════════════════
# Band Encoder: n_bands → 16 → 8 → 1 (Synthetic VI bottleneck)
# ══════════════════════════════════════════════════════════════════════════
BandEncoder <- nn_module(
  classname = "BandEncoder",
  initialize = function(n_bands = 5, h1 = 16, h2 = 8) {
    self$fc1  <- nn_linear(n_bands, h1)
    self$fc2  <- nn_linear(h1, h2)
    self$fc3  <- nn_linear(h2, 1)      # bottleneck = Synthetic VI (linear out)
    self$drop <- nn_dropout(0.1)
  },
  forward = function(x) {
    x <- x %>% self$fc1() %>% nnf_relu() %>% self$drop()
    x <- x %>% self$fc2() %>% nnf_relu() %>% self$drop()
    self$fc3(x)
  }
)

# ══════════════════════════════════════════════════════════════════════════
# Full predictor: encoder → Synthetic VI → prediction head → trait
# ══════════════════════════════════════════════════════════════════════════
SyntheticVIModel <- nn_module(
  classname = "SyntheticVIModel",
  initialize = function(n_bands = 5, pred_h = 8) {
    self$encoder  <- BandEncoder(n_bands)
    self$pred_fc1 <- nn_linear(1, pred_h)
    self$pred_fc2 <- nn_linear(pred_h, 1)
    self$drop     <- nn_dropout(0.1)
  },
  forward = function(bands) {
    vi <- self$encoder(bands)                            # (N, 1)
    vi %>% self$pred_fc1() %>% nnf_relu() %>% self$drop() %>% self$pred_fc2()
  },
  get_synthetic_vi = function(bands) {
    self$encoder(bands)
  }
)

5 Training

Two CV scenarios are run in parallel:

  • LOEO: holds out one environment at a time, trains on the other four.
  • Within-env 5-fold CV: within each environment, GIDs are assigned to 5 folds; the model trains on 4 folds and predicts the 5th.

In both, the band scaler (0–1 min–max) and trait scaler (z-score) are fit on training data only and applied to the test fold — model_data_raw (unscaled) is passed to the CV functions so scaling is done internally.

After CV, a final model trained on all data is used to extract the Synthetic VI formula.

# ── Helper: build band + label tensors from a pre-scaled data frame ────────
make_tensors <- function(df, trait) {
  df_sub <- df[!is.na(df[[trait]]), ]
  list(
    bands = torch_tensor(as.matrix(df_sub[, BAND_NAMES, drop = FALSE]),
                         dtype = torch_float()),
    y     = torch_tensor(df_sub[[trait]], dtype = torch_float())$unsqueeze(2)
  )
}

# ── Scale df using stats from train_df; return scaled df + back-transform params
scale_from_train <- function(df, train_df, trait) {
  band_sc  <- map(BAND_NAMES, function(b)
    list(mn = min(train_df[[b]], na.rm = TRUE),
         mx = max(train_df[[b]], na.rm = TRUE))) %>% set_names(BAND_NAMES)
  trait_mn <- mean(train_df[[trait]], na.rm = TRUE)
  trait_sd <- sd(train_df[[trait]],   na.rm = TRUE)
  for (b in BAND_NAMES) {
    sc <- band_sc[[b]]
    df[[b]] <- (df[[b]] - sc$mn) / (sc$mx - sc$mn + 1e-8)
  }
  df[[trait]] <- (df[[trait]] - trait_mn) / (trait_sd + 1e-8)
  list(df = df, trait_mn = trait_mn, trait_sd = trait_sd)
}

# ── Core training loop ─────────────────────────────────────────────────────
train_model <- function(train_df, trait,
                        epochs = 300, lr = 3e-3, batch_size = 64,
                        seed = NULL, verbose = FALSE) {
  if (!is.null(seed)) torch_manual_seed(seed)
  tensors <- make_tensors(train_df, trait)
  n       <- tensors$bands$size(1)
  if (n < 10) return(NULL)

  model <- SyntheticVIModel(n_bands = length(BAND_NAMES), pred_h = 8)
  opt   <- optim_adam(model$parameters, lr = lr, weight_decay = 1e-4)
  sched <- lr_step(opt, step_size = 100, gamma = 0.5)

  model$train()
  for (ep in seq_len(epochs)) {
    idx <- sample(n)
    for (i in seq(1, n, by = batch_size)) {
      bi <- idx[i:min(i + batch_size - 1, n)]
      opt$zero_grad()
      pred <- model(tensors$bands[bi, , drop = FALSE])
      loss <- nnf_mse_loss(pred, tensors$y[bi, , drop = FALSE])
      loss$backward()
      nn_utils_clip_grad_norm_(model$parameters, 1.0)
      opt$step()
    }
    sched$step()
    if (verbose && ep %% 100 == 0)
      cat(sprintf("  ep %d | loss %.4f\n", ep, loss$item()))
  }
  model$eval()
  model
}

# ── LOEO: Leave-One-Environment-Out CV ────────────────────────────────────
# Takes raw (unscaled) data. Scaler fit on training environments only.
loeo_cv <- function(data_raw, trait, n_reps = 3,
                    epochs = 300, lr = 3e-3, batch_size = 64) {
  envs <- sort(unique(as.character(data_raw$Env)))
  map_dfr(envs, function(held_env) {
    train_raw <- data_raw %>% filter(Env != held_env, !is.na(.data[[trait]]))
    test_raw  <- data_raw %>% filter(Env == held_env,  !is.na(.data[[trait]]))
    if (nrow(train_raw) < 10 || nrow(test_raw) < 2) return(NULL)

    sc_out   <- scale_from_train(train_raw, train_raw, trait)
    train_sc <- sc_out$df
    trait_mn <- sc_out$trait_mn
    trait_sd <- sc_out$trait_sd
    test_sc  <- scale_from_train(test_raw, train_raw, trait)$df

    map_dfr(seq_len(n_reps), function(rep_i) {
      model <- tryCatch(
        train_model(train_sc, trait, epochs = epochs, lr = lr,
                    batch_size = batch_size, seed = rep_i * 100L),
        error = function(e) { message(held_env, " rep", rep_i, ": ", e$message); NULL }
      )
      if (is.null(model)) return(NULL)
      te_ten <- make_tensors(test_sc, trait)
      with_no_grad({
        pred_z <- model(te_ten$bands)$squeeze()$to(device = "cpu")
      })
      pred <- as.numeric(pred_z) * trait_sd + trait_mn
      obs  <- as.numeric(te_ten$y$squeeze()) * trait_sd + trait_mn
      tibble(held_env = held_env, rep = rep_i, n_test = length(obs),
             r    = cor(pred, obs, use = "complete.obs"),
             rmse = sqrt(mean((pred - obs)^2, na.rm = TRUE)))
    })
  })
}

# ── Within-environment 5-fold CV (split by GID) ───────────────────────────
# Takes raw (unscaled) data. GIDs assigned to folds; scaler fit on train GIDs.
within_env_cv <- function(data_raw, trait, n_folds = 5, n_reps = 3,
                          epochs = 300, lr = 3e-3, batch_size = 64) {
  envs <- sort(unique(as.character(data_raw$Env)))
  map_dfr(envs, function(env_i) {
    env_df <- data_raw %>% filter(Env == env_i, !is.na(.data[[trait]]))
    gids   <- unique(env_df$GID)
    if (length(gids) < n_folds) {
      message("Env ", env_i, ": ", length(gids), " GIDs < n_folds — skipping")
      return(NULL)
    }
    map_dfr(seq_len(n_reps), function(rep_i) {
      set.seed(rep_i * 77L)
      shuffled <- sample(gids)
      fold_id  <- rep(seq_len(n_folds), length.out = length(shuffled))
      gid_fold <- tibble(GID = shuffled, fold = fold_id)

      map_dfr(seq_len(n_folds), function(f) {
        test_gids   <- gid_fold$GID[gid_fold$fold == f]
        train_raw_f <- env_df %>% filter(!GID %in% test_gids)
        test_raw_f  <- env_df %>% filter( GID %in% test_gids)
        if (nrow(train_raw_f) < 10 || nrow(test_raw_f) < 2) return(NULL)

        sc_out   <- scale_from_train(train_raw_f, train_raw_f, trait)
        train_sc <- sc_out$df
        trait_mn <- sc_out$trait_mn
        trait_sd <- sc_out$trait_sd
        test_sc  <- scale_from_train(test_raw_f, train_raw_f, trait)$df

        model <- tryCatch(
          train_model(train_sc, trait, epochs = epochs, lr = lr,
                      batch_size = batch_size, seed = rep_i * 100L + f),
          error = function(e) NULL
        )
        if (is.null(model)) return(NULL)
        te_ten <- make_tensors(test_sc, trait)
        with_no_grad({
          pred_z <- model(te_ten$bands)$squeeze()$to(device = "cpu")
        })
        pred <- as.numeric(pred_z) * trait_sd + trait_mn
        obs  <- as.numeric(te_ten$y$squeeze()) * trait_sd + trait_mn
        tibble(Env = env_i, fold = f, rep = rep_i, n_test = length(obs),
               r    = cor(pred, obs, use = "complete.obs"),
               rmse = sqrt(mean((pred - obs)^2, na.rm = TRUE)))
      })
    })
  })
}

9 Synthetic VI Formula

The Synthetic VI is defined by the three learned linear layers of the encoder:

\[ \mathbf{h}_1 = \text{ReLU}(\mathbf{W}_1 \mathbf{x} + \mathbf{b}_1), \quad \mathbf{h}_2 = \text{ReLU}(\mathbf{W}_2 \mathbf{h}_1 + \mathbf{b}_2), \quad \text{SynVI} = \mathbf{w}_3^\top \mathbf{h}_2 + b_3 \]

where \(\mathbf{x} = [\text{blue, green, red, red\_edge, nir}]^\top\) are the min-max scaled band values. The tables below show the extracted weights for each trait’s full-data model.

# LOGO linear regression benchmark for each standard VI × trait
linreg_logo_cv <- function(data, vi, trait) {
  map_dfr(group_levels, function(held_out) {
    tr <- data %>% filter(group != held_out,
                          !is.na(.data[[vi]]), !is.na(.data[[trait]]))
    te <- data %>% filter(group == held_out,
                          !is.na(.data[[vi]]), !is.na(.data[[trait]]))
    if (nrow(tr) < 10 || nrow(te) < 2) return(NULL)
    fit  <- lm(as.formula(paste(trait, "~", vi)), data = tr)
    pred <- predict(fit, newdata = te)
    tibble(held_out = held_out,
           r = cor(pred, te[[trait]], use = "complete.obs"))
  }) %>%
  summarise(mean_r = mean(r, na.rm = TRUE)) %>%
  pull(mean_r)
}
